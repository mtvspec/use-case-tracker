--------------------------------------------------------------------------------
-- Actor (A)
--------------------------------------------------------------------------------
An actor defines a role that a user can play when interacting with the system.
A user can either be an individual or another system.
Actors have a name and a brief description, and they are associated to the
use cases with which they interact.
Actor - an individual or other system that interacts with system to achieve
particular goal by use case of system.

-- Use Case (UC)
--------------------------------------------------------------------------------
A use case is all the ways of using a system to achieve a particular goal for
particular user.
Precision Level 0. Scope establishment (What's the Scope?)
Scope representation techniques
1) Use case table
  Subject
  a) Actor
  b) Goal (use case)
  c) Level
  d) Description (use-case summary)
2) Use case model
  a) Subject
  b) Actor
  c) Use case
Precision Level 1. Header information gathering (Who Wants What?)
a) goal
b) actor (Stakeholders and Interests)
c) description
d) code
e) trigger
f) pre-conditions
g) post-conditions

--- Flow (F)
--------------------------------------------------------------------------------
A description of some full or partial path through a use-case narrative. There
is always at least a basic flow, and where may be alternative flows.
--------------------------------------------------------------------------------
---- Basic Flow (BF)
--------------------------------------------------------------------------------
A description of the normal, expected path through the use case. This is the
path taken by most of the users most of the time; it is the most important part
of use-case narrative.
--------------------------------------------------------------------------------
---- Alternative Flow (AF)
--------------------------------------------------------------------------------
A description of variant or optional behaviour as part of a use-case narrative.
Alternative flows are defined relative to the use case's basic flow.
--------------------------------------------------------------------------------
--- Use Case Narrative (UCN)
--------------------------------------------------------------------------------
A description of a use case that tells the story of how the system and its
actors work together to achieve a particular goal. It includes a sequence of
actions (including variants) that a system and its actor can perform to achieve
a goal.
--------------------------------------------------------------------------------
-- Use Case Diagram (UCD)
--------------------------------------------------------------------------------
A diagram showing a number of actors and use cases, and their relationships.
--------------------------------------------------------------------------------
-- Use Case Model (UCM)
--------------------------------------------------------------------------------
A model of all of the useful ways to use a system, and the value that it will
provide. A use-case model is primarily made up of a set of actors and set of
use cases, and diagrams illustrating their relationships.
--------------------------------------------------------------------------------
-- Use Case Slice (SL)
--------------------------------------------------------------------------------
A use-case slice is one or more stories selected from a use case to form a work
item that is of clear value to the customer.
--------------------------------------------------------------------------------
Story (ST)
--------------------------------------------------------------------------------
A way of using a system that is of value to a user or other stakeholder.
In the Use-Case 2.0 a story is described by part of the use-case narrative,
one or more flows and special requirements, and one or more test cases.
