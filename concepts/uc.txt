Create Use Case
--------------------------------------------------------------------------------
Пользователь создает новый UC. Указывает наименование, описание, триггер,
предварительные условия и пост-условия (гарантии). Создает основной сценарий,
альтернативные сценарии, исключительные ситуации, вложенные сценарии и
расширения. Ищет и выбирает (создает) бизнес-правила. Каждый UC состоит
из сценариев. Каждый сценарий состоит из шагов. Каждый шаг выполняется
актором или системой. Актор и система осуществляют шаги поочередно.
Некоторые шаги могут заменяться шагами альтернативных сценариев.
Сценарии может быть расширен дополнительными шагами. В UC можно включать
сценарии из других UC. Результаты обработки каждого шага могут вызывать шаги
из других сценариев других UC. У каждого UC есть основной актор и могут быть
дополнительные. Основной сценарии всегда успешный. Сценарий может закончиться
успехом (цель актора достигнута), а может закончиться неудачей (цель актора
недостигнута). Цель актора может быть достигнута несколькими UC.

Основной сценарий:
1. Пользователь создает UC.
2. Система отображает интерфейс создания UC.
3. Пользователь заполняет атрибуты UC:
- наименование
Пользователь сохраняет UC.
4. Система создает UC.

Исключение - это событие, которое приводит к ошибке, в случае отсутствия
обработчика данной ошибки. То есть, исключение - это шаг, который заменяет
шаг основного или альтернативного сценариев.
Обработчик исключения (ошибки) - это действие системы, которое либо требует
повторного выполнения шага с корректными параметрами, в котором совершена
ошибка, либо выполнения определенного шага, обрабатывающего данную
исключительную ситуацию.

Card attributes
--------------------------------------------------------------------------------
- creator (user)
- author (stakeholder - current)
- assignee (project member - current)
- name
- description
- state (open|closed)
- status (status list)
--------------------------------------------------------------------------------
Card tags
--------------------------------------------------------------------------------
- BR (Bug Report)
- CR (Change Request)
--------------------------------------------------------------------------------
Card status list
--------------------------------------------------------------------------------
- Created (when card is created)
- Scoped (when card's goal established)
- Prepared (when card's specs prepared)
- To Do (when card's specs and task accepted)
- Doing (when card specs in work)
- Done (when card is done)
- Verified (when card work is tested and accepted by tester)
- Accepted (when card work is tested and accepted by stakeholder)
- On hold (when card work is on hold)
- Duplicate (when card is duplicate)
- Rejected (when card is rejected)
