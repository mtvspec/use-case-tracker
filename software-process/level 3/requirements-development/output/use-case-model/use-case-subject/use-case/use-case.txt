-- Use Case (object)
--============================================================================--
--------------------------------------------------------------------------------
-- Purpose
--------------------------------------------------------------------------------
The purpose of this object is to store data about which goal and how can be
achieved by use case.
--------------------------------------------------------------------------------
-- Definition
--------------------------------------------------------------------------------
a) A sequence of actions a system performs that yields an observable result
of value to a particular user.
b) That specific behavior of a system, which participates in a collaboration
with a user to deliver something of value for that user.
c) The smallest unit of activity that provides a meaningful result to the user.
d) The context for a set of related requirements.
e) A conversation of actor(s) and system.
f) A agreement about system behavior.

-- Lifecycle
--------------------------------------------------------------------------------
- New
When the use case created
- Goal established
When the goal of the use case has been established
- Story structure understood
When the structure of the use-case narrative has been understood enough for the
team to start work identifying and implementing the first use-case slices
- Simplest story fulfilled
When the system fulfills the simplest story that allows a user to achieve the
goal
- Sufficient stories fulfilled
When the system fulfills enough of the stories to provide a usable solution
- All stories fulfilled
When the system fulfills all the stories told by the use case
- Accepted
When the use case accepted by its owner
--------------------------------------------------------------------------------
-- Code (attribute)
--------------------------------------------------------------------------------
This attribute stores use-case's code.
Constraints:
1) Use case code is unique in the use case model.
2) Use case code length is 3 digits.
3) Use case code prefix format: 'UC'
3) Use case code full format example: 'UC-003'.
--------------------------------------------------------------------------------
-- Name (attribute)
--------------------------------------------------------------------------------
This attribute stores use-case's name.
Constraints:
1) Attribute is unique in the use case model.
2) Attribute length is 1000 characters.
3) Attribute datatype is text.
Rules:
1) Attribute value must represent actor's goal achieved by use case.
--------------------------------------------------------------------------------
-- Type (attribute)
--------------------------------------------------------------------------------
This attribute stores type id.
Types:
1) Business use-case
2) System use-case
3) System (internal) use-case
--------------------------------------------------------------------------------
-- Level (attribute)
--------------------------------------------------------------------------------
This attribute stores level id.
Levels:
1) Black Box
2) White Box
--------------------------------------------------------------------------------
-- Summary (attribute)
--------------------------------------------------------------------------------
This attribute stores use case's summary.
Constraints:
1) Attribute is unique in the use case model.
2) Attribute length is 4000 characters.
3) Attribute datatype is text.
